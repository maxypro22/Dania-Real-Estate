5. REMAINING ITEMS (NOT code — owner decision / asset production)
--------------------------------------------------------------------------------
5.1 BLOG  — INTENTIONALLY NOT ADDED
    The spec contains one line: "Blog category page (add to footer section)".
    However the spec's own canonical URL list does NOT include a /blog/ URL,
    and no blog page, blog content, or blog blueprint exists anywhere in the
    spec. Adding a footer "Blog" link would therefore create a dead link and
    REDUCE conformance/quality. Recommendation: supply blog content + a /blog/
    blueprint, then it can be added in minutes (route + footer link + SEO entry).

5.2 PHOTOGRAPHY (Gallery + Home)  — NEEDS REAL ASSETS
    The Gallery and parts of the Home page use remote Pexels stock images, not
    spec-named local property photos. The structural requirements (alt text,
    layout, responsive behaviour) are met, but genuine Dania property
    photography cannot be fabricated by code. Drop real .webp files into public/
    using the spec naming convention and swap the Pexels URLs to local paths.

5.3 (Optional, recommended) PRERENDER / SSG
    The site is a client-rendered SPA, so title/meta/canonical/JSON-LD are
    injected by JS. Google renders JS and will see them, but for maximum
    crawler reliability and instant social-card previews, a prerender/SSG step
    (e.g. vite-plugin-ssg or a prerender build) would bake these into the static
    HTML per route. Not required for spec conformance; flagged as best practice.
